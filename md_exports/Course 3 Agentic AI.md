Be Careful between AI agents and Classic AI Bots
An AI agent possesses four core properties:
1.Perception — reads context (documents, APIs, UI, sensors)
2.Reasoning — plans multi-step actions toward a goal
3.Action — calls tools, writes code, sends emails, queries DBs
4.Memory — retains context across steps (short-term) and sessions (long-term)

Agentic AI:
The Trust Problem – What Can Go Wrong ?
• Hallucination with action: Agent acts on false information
• Prompt injection: Malicious content hijacks agent behavior
• Scope creep: Agent takes unintended actions (e.g., deletes files)
• Runaway loops: Agent stuck in infinite retry cycles
• Data leakage: Agent exposes sensitive information through tool calls
The "Human in the Loop" Spectrum

Agentic AI:
Governance and Regulatory 
Governance Framework for Agentic AI
1.Define agent scope: What can it access? What actions are permitted?
2.Establish audit trails: Every agent action must be logged
3.Set kill switches: Mechanisms to halt agents instantly
4.Implement sandboxing: Test before production deployment
5.Define accountability: Who is responsible when an agent fails?
6.Monitor continuously: Behavioral drift detection
 Regulatory Landscape
•EU AI Act (2024): Risk-tiered regulation — autonomous agents in high-risk domains face 
strict requirements
•GDPR implications: Agent memory and data retention
•SEC guidance: AI use in financial advice and trading
•Emerging: Agent identity standards, liability frameworks

Agentic RAG:
Overview
Agentic RAG gives the retrieval process a brain. Instead of one fixed retrieve → generate step, an agent 
decides how, when, where, and how many times to retrieve — and can act on what it finds.

## Page 41

Agentic RAG :
Different Architectures 
1. Corrective RAG (CRAG) Agent evaluates retrieved documents → if low confidence, triggers a web search 
fallback.
Use case: Compliance agents that need up-to-date regulatory information.
2. Self-RAG Agent decides at every step: "Do I need to retrieve? Is this retrieved doc relevant? Is my answer 
supported?"
Use case: Medical or legal Q&A where grounding is critical.
3. Adaptive RAG Routes queries dynamically: simple questions → direct LLM answer; complex ones → full 
retrieval pipeline.
Use case: Customer service bots handling both FAQs and complex account issues.
4. Multi-Agent RAG Specialized agents retrieve from different sources (internal DB, web, CRM) and an 
orchestrator synthesizes results.
Use case: Investment research combining filings, news, and internal analyst notes.
